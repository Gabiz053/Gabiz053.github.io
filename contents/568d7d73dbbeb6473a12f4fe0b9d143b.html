Description

Satisfacción de requisitos no funcionales Eficiencia de desempeño: El sistema debe presentar un tiempo de respuesta menor a 1 segundo para reproducir canciones de forma continuada, por ejemplo, al cargar una playlist el tiempo de respuesta podrá ser superior a 1 segundo, pero que sea inferior a 4 segundos (la primera canción). Decisión: Se permite la escalabilidad partiendo de dos servidores hasta llegar a un número de estos suficiente como para permitir el buen funcionamiento de este, empleando el balanceador de carga.

Inicialmente el sistema deberá soportar al menos 10000 usuarios conectados a la vez. Decisión: Se permite la escalabilidad partiendo de dos servidores hasta llegar a un número de estos suficiente como para permitir el buen funcionamiento de este, empleando el balanceador de carga.

Fiabilidad: El sistema ha de presentar una disponibilidad del 99%. Decisión: Se parte de dos servidores, uno principal y otro de respaldo por si el primero fallase.

El sistema ha de poder recuperarse de una caída completa en menos de 30 minutos. Decisión: Se parte de dos servidores, uno principal y otro de respaldo por si el primero fallase.

Seguridad: El sistema ha de proteger los datos personales de los usuarios. Decisión: Se ha creado una base de datos (cifrado.sql) para proteger y aislar los datos de usuarios (incluyendo sus datos de pago) del resto de datos del programa.

El sistema deberá registrar las operaciones realizadas que impliquen pagos. Decisión: Se ha creado una base de datos (cifrado.sql) para proteger y aislar los datos de usuarios (incluyendo sus datos de pago) del resto de datos del programa.

Mantenibilidad: El servidor en el que se vaya a hostear la aplicación deberá facilitarnos opciones para escalar los recursos de este. Decisión: Se parte del uso de dos servidores (uno principal y otro de respaldo) pero pueden escalarse estos hasta el número que se requieran en base a nuestras necesidades Portabilidad: El sistema ha de poder ejecutarse en sistemas operativos Windows, Linux, osx, Android e IOS. Decisión: Se ha implementado una aplicación móvil para los dispositivos Android e IOS, y un cliente web para los dispositivos Windows, Linux y OSX.
